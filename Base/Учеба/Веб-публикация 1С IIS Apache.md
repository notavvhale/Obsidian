
Первым делом понадобится сервер для публикации на IIS.
В моём случае это Windows Server 2019.
Далее ставим ломаный дистрибутив 1С и качаем конфигурацию releases.1c.ru. ([[Учётка 1С]])
==Или==
Вот торренты на готовую комбинацию конфигурация+платформа (я не проверял)
[1С Бухгалтерия предприятия 2.0 Базовая + Платформа 8.3.18.1289](https://rutracker.org/forum/viewtopic.php?t=5436377)
[1С Бухгалтерия предприятия 2.0 + Платформа 8.3.20.1674](https://rutracker.org/forum/viewtopic.php?t=5436377)

==Обязательно== надо при установке поставить пункт ==ВЕБ МОДУЛИ==
![[Pasted image 20260204092835.png]]

После того как установили, заходим на сайт апач для windows
https://www.apachelounge.com/download/ качаем пакет вот этот
![[Pasted image 20260204093712.png]]
Распаковать эту папку по этому пути и зайти в conf/httpd.conf раскоментить строчку ServerName и заменить http://example.com на localhost
![[Pasted image 20260204094246.png]]
Открываем командную строку от администратора и переходим в папку **bin** каталога, куда распаковали apache — в нашем случае:

cd C:\Apache24\bin

Запускаем установку apache как сервиса Windows:

httpd.exe -k install

Мы должны увидеть:

Installing the 'Apache HTTP Server' service  
 The 'Apache HTTP Server' service is successfully installed.  
 Testing httpd.conf....  
 Errors reported here must be corrected before the service can be started.

Ещё нужно зайти в службы и запустить Apache2.4
==Теперь 1С увидит Apache2.4 сервер и будет возможно опубликовать базу.==

Чтобы появилась возможность публикации на IIS нужно установить роли на сервер.![[Pasted image 20260204112234.png]]
![[Pasted image 20260204170507.png]]
==Теперь 1С увидит IIS сервер и будет возможно опубликовать базу.==

Переходим к самой публикации базы.
Создаём базу по конфигурации и заходим в конфигуратор.
Жмём Администрирование>Публикация на веб-сервере.
![[Pasted image 20260204112518.png]]
Тут выбираем апач потому что сейчас он запущен.
Надо ещё обязательно выбрать каталог откуда публикуется база.
==`C:\inetpub\wwwroot\DemoStateAccounting2\`== - дефолтный путь для IIS
==`C:\Apache2.4\htdocs\Base1\DemoStateAccounting2\`== - не дефолтный путь, надо руками прописывать в 1С и создавать папку.
Если запущен IIS запустить апач не получится. Надо менять порт.
Когда всё сделали жмём опубликовать.
С апачем какой-то дроч, я просто несколько раз перепубликовывал и в итоге заработало.
База будет доступна по пути http://localhost/(название_базы)
