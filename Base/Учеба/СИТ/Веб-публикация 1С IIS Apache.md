https://www.apachelounge.com/download/ качаем пакет вот этот

Распаковать эту папку по этому пути, зайти в conf/httpd.conf раскомментить строчку ServerName и заменить http://example.com на localhost

Открываем командную строку от администратора и переходим в папку **bin** каталога, куда распаковали apache — в нашем случае:

Запускаем установку apache как сервиса Windows:

Ещё нужно зайти в службы и запустить Apache2.4

Чтобы появилась возможность публикации на IIS нужно установить роли на сервер.![[Pasted image 20260204112234.png]]

==Теперь 1С увидит IIS сервер и будет возможно опубликовать базу.==

Переходим к самой публикации базы.
Создаём базу по конфигурации и заходим в конфигуратор.
Жмём Администрирование>Публикация на веб-сервере.
![[Pasted image 20260204112518.png]]
Тут выбираем апач потому что сейчас он запущен.
Надо ещё обязательно выбрать каталог откуда публикуется база.
==`C:\inetpub\wwwroot\DemoStateAccounting2\`== - дефолтный путь для IIS
==`C:\Apache2.4\htdocs\Base1\DemoStateAccounting2\`== - не дефолтный путь, надо руками прописывать в 1С и создавать папку.
Если запущен IIS запустить апач не получится. Надо менять порт.
Когда всё сделали жмём опубликовать.
С апачем какой-то дроч, я просто несколько раз перепубликовывал и в итоге заработало.
База будет доступна по пути http://localhost/(название_базы)

# HTTPS

Чтобы опубликовать базу с SSL сертификатом, нужно его выпустить.
Для выпуска сертификата понадобятся два инструмента. 
OpenSSL - https://slproweb.com/products/Win32OpenSSL.html
и 
mkcert - https://github.com/FiloSottile/mkcert/releases.

Сначала создаём сертификат с помощью mkcert.
Через Powershell нужно запустить скачанный файл. Можно переназвать его mkcert.exe и закинуть в C:\ .
Далее открываем Powershell и пишем сначала
`cd C:\`
а затем пишем
`.\mkcert.exe -install`
Должно получиться так.
![[Pasted image 20260326140802.png]]
После установки создаём сертификат.
`.\mkcert.exe 192.168.1.10 mysite.local`
После создания, появятся файлики .pem расширения.

Далее нужно переформатировать эти файлы в понятное IIS .pfx расширение.
Для этого качаем OpenSSL и устанавливаем этот файлик просто далее далее.
Запускаем OpenSSL CLI
![[Pasted image 20260327124644.png]]
![[Pasted image 20260327124733.png]]
Заходим в папку где лежат сертификаты .pem
`cd C:\`
Далее пишем команду
`openssl pkcs12 -export -out cert.pfx -inkey 192.168.1.10+1-key.pem -in 192.168.1.10+1.pem`
вместо "192.168.1.10+1-key.pem" подставляем своё имя файла.
Далее вводим пароль например `1234`
Сформируется файлик с нужным нам расширением .pfx.
![[Pasted image 20260327125317.png]]

Далее заходим в IIS
Добавляем привязку(bindings) 443 порт
![[Pasted image 20260326135956.png]]
![[Pasted image 20260327125650.png]]

Готово, должно работать.

Если не работает, можно перезапустить iis
`iisreset`
